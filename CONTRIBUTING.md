# Contributing to Weather Prediction ML

Thank you for your interest in contributing! We welcome contributions from everyone.

## How to Contribute

### 1. Report Bugs
- Open an Issue on GitHub
- Describe the bug clearly
- Include steps to reproduce
- Show expected vs actual behavior

### 2. Suggest Enhancements
- Open an Issue to discuss your idea first
- Explain why the enhancement would be useful
- Provide examples or use cases

### 3. Submit Code Changes

#### Fork and Clone
```bash
git clone https://github.com/YOUR_USERNAME/weather-prediction-machine-learning.git
cd weather-prediction-machine-learning
```

#### Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

#### Make Your Changes
- Write clear, readable code
- Add comments for complex logic
- Test your changes thoroughly
- Follow PEP 8 style guide

#### Commit Your Changes
```bash
git add .
git commit -m "Add your descriptive message here"
```

#### Push and Create Pull Request
```bash
git push origin feature/your-feature-name
```

## Code Style Guidelines

### Python Code
- Follow PEP 8
- Use meaningful variable names
- Add docstrings to functions
- Keep functions small and focused

### Commit Message Format
```
[TYPE] Brief description

Detailed explanation of changes
- Point 1
- Point 2
```

Examples:
- `[FEATURE] Add neural network model`
- `[BUG] Fix data normalization`
- `[DOCS] Update README`

## Development Setup

```bash
# Create virtual environment
python -m venv venv

# Activate virtual environment
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

## Pull Request Checklist

- [ ] Code follows PEP 8
- [ ] Tests pass locally
- [ ] Documentation is updated
- [ ] Commit messages are clear
- [ ] No large files added
- [ ] No sensitive data included
